## Notes & Edge Cases

- **Mobile Viewport**: 393px width targets iPhone 14/15 Pro. Design uses a mobile-first layout with bottom tab navigation instead of sidebar.
- **Status Bar**: Shows iOS status bar with "9:41" time and system icons. In implementation, this is handled by the native browser/OS — do not render it in the web app.
- **Bottom Tab Bar**: 4 tabs (Home, Assignments, Library, AI Toolkit) replace the desktop sidebar. Needs safe-area-inset handling for devices with home indicator.
- **Safe Area**: Use `env(safe-area-inset-bottom)` for bottom padding to avoid the home indicator on notched iPhones.
- **Touch Targets**: All interactive elements should meet 44×44px minimum touch target size (Apple HIG).
- **Desktop Counterpart**: `0-State-screen` (1440×780px) — same content, different layout.
- **WebView**: The "web-to-figma.design" text in the status bar area suggests this may be viewed in a WebView or is an artifact from the Figma mobile frame template.
