## Notes & Edge Cases

- **First-Time User Experience**: This screen appears only when the user has zero assignments. Once the first assignment is created, this screen should never show again for that user.
- **Illustration**: The empty state features a custom illustration (papers/documents theme with colors `#011625`, `#CCC5D8`, `#417BA4`, `#17CB9E`). Export as SVG from Figma for crisp rendering.
- **CTA Button**: "Create Your First Assignment" is the primary action — should be visually prominent and centered. The text differs from the sidebar's "Create Assignment" to emphasize the first-time experience.
- **Responsive**: Desktop variant at 1440×780px. Mobile equivalent is `Dashboard-Mobile-Empty` (393×857px) with a similar layout adapted for small screens.
- **Sidebar**: Identical sidebar component shared across all desktop screens. "Assignments" is the active nav item.
- **Loading State**: Consider adding a skeleton/shimmer state while checking if assignments exist (to avoid flash of empty state).
- **Accessibility**: CTA button is the primary focus target on page load. Illustration needs descriptive `alt` text.
- **Hidden Layers**: `Close icon` vector found — likely unused on this screen, may be a leftover from another state.
