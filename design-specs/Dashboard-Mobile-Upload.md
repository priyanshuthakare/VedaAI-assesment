## Notes & Edge Cases

- **Mobile File Upload**: On mobile, "Browse Files" should trigger `<input type="file" accept="image/*">` which opens the native file/camera picker with options for Camera, Photo Library, and Files.
- **Long Scrollable Form**: At 1397px height, the form is significantly taller than viewport. Ensure smooth scrolling with `-webkit-overflow-scrolling: touch`.
- **Sticky Form Actions**: "Previous" and "Next" buttons should be sticky at the bottom above the tab bar, always visible regardless of scroll position.
- **Keyboard Handling**: When input is focused, the virtual keyboard pushes content. Use `visualViewport` API or `resize` event to adjust layout.
- **Date Picker**: On iOS/Android, `<input type="date">` triggers the native date picker. No custom date picker component needed.
- **Number Inputs**: Marks fields should use `inputmode="numeric"` to show the numeric keyboard on mobile.
- **Desktop Counterpart**: `Upload-Material---Selector` (1440×1340px).
- **Safe Area**: Bottom actions and tab bar need safe-area-inset handling for devices with home indicator.
