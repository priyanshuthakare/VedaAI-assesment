## Notes & Edge Cases

- **File Upload Constraints**: Accepts JPEG and PNG only, max 10MB per file. The UI should show upload progress and error states for oversized or wrong-format files.
- **Drag & Drop**: The dashed border upload zone supports both click-to-browse and drag-and-drop. Needs `dragover`, `dragleave`, and `drop` event handling with visual feedback (border color change to `#171717`).
- **Date Picker**: The "DD-MM-YYYY" format suggests localized date display. Consider using a date library (date-fns/dayjs) for formatting.
- **Question Type Management**: Users can add/remove question types dynamically. Each row has a name, marks input, and delete button. Total marks should auto-calculate.
- **Form Validation**: "Next" button should validate: file uploaded, due date set, at least one question type with marks > 0.
- **Responsive**: This is the desktop (1440px) variant. The mobile equivalent is `Dashboard-Mobile-Upload` (393px width).
- **Hidden Layers**: `icon_line/Arrow_Right`, `Rectangle 15`, `Settings` (×2) — some sidebar icons have hidden vector sub-layers; use the visible composite instead.
- **Backdrop Blur**: Top nav bar uses `background: #FFFFFFBF` (white at 75% opacity) with likely backdrop-filter blur — verify in Figma.
- **Hover/Focus States**: Not captured in static export. Recommended: darken bg by 5% on hover for white buttons, reduce opacity to 0.9 for dark buttons, add `ring-2 ring-offset-2` focus ring.
- **Animations**: Consider slide-in transition for the form sections and fade for upload state changes.
- **Accessibility**: Upload zone needs `role="button"` and keyboard support (Enter/Space to trigger). Date input needs `aria-label`. All nav items need `aria-current` for active state.
