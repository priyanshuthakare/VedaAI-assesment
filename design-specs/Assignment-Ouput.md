## Notes & Edge Cases

- **AI-Generated Content**: The question paper is generated by AI. The "Certainly, Lakshya!" message is the AI's conversational response before the formal paper.
- **Paper Format**: Follows a standard Indian school examination paper format — school name, subject, class, time, marks, student fields, instructions, then sections with questions.
- **Difficulty Tags**: Questions are tagged with `[Easy]`, `[Moderate]`, `[Hard]` and marks `[2 Marks]`. These should be visually distinct (bold, possibly color-coded).
- **Section Organization**: Paper is divided into sections (A, B, C...) with different question types and mark allocations.
- **Print Styling**: Needs `@media print` CSS to strip sidebar, nav, AI message, and action buttons. Only the paper content should print.
- **PDF Export**: "Download PDF" should use a library like `react-to-print` or `html2pdf.js` to generate a clean PDF of the paper.
- **Long Content**: Paper can be very long (1715px height in design). Content area needs scrolling. Consider pagination for print.
- **Responsive**: Desktop at 1440px. Mobile equivalent is `Dashboard-Mobile-Output` (393px) with the paper taking full width.
- **Regeneration**: "Regenerate" should send back to the AI with the same parameters to produce alternative questions.
- **Editable Output**: Consider making individual questions editable in-place so teachers can fine-tune the AI output.
- **Active Sidebar**: "AI Teacher's Toolkit" appears to be the active sidebar item on this screen (different from assignments).
