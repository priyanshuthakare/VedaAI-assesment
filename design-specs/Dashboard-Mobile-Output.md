## Notes & Edge Cases

- **Very Long Page**: At 2821px, this is the tallest screen. Content must scroll smoothly. Consider lazy rendering for performance.
- **Reading Experience**: This is primarily a reading view. Font size (14px) and line-height (1.6) are optimized for mobile reading.
- **Share via Native**: On mobile, the share button should use the Web Share API (`navigator.share()`) to trigger the native share sheet.
- **PDF Download**: On mobile, downloading a PDF may open it in the browser's PDF viewer. Consider using the native share for "Save to Files" on iOS.
- **Offline Support**: Consider caching the generated paper for offline viewing (Service Worker).
- **Desktop Counterpart**: `Assignment-Ouput` (1440×1715px).
- **No Bottom Tab Bar**: This screen appears to not have the bottom tab bar, focusing entirely on the paper content.
- **Pinch to Zoom**: Allow pinch-to-zoom for reading the question paper text on small screens.
