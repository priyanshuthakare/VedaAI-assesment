# Database Schemas (MongoDB)

The backend uses **Mongoose** to interact with MongoDB. The data model is strictly separated into two collections: one for the user's input configuration, and one for the AI's generated output.

## 1. Assignment Schema
Stores the configuration rules submitted by the user on the frontend.
- **File:** `src/models/Assignment.ts`
- **Fields:**
  - `status`: String (`pending`, `completed`, `failed`). Defaults to `pending`.
  - `input`: Object containing the raw form data.
    - `topic`: String
    - `totalQuestions`: Number
    - `marksPerQuestion`: Number
    - `questionTypes`: Array of objects `{ type, count, marksEach }`
    - `difficulty`: Object `{ easy, medium, hard }`
    - `dueDate`: String (optional)
    - `instructions`: String (optional)
  - `createdAt` / `updatedAt`: Timestamps

## 2. Result Schema
Stores the structured JSON paper generated by the AI model.
- **File:** `src/models/Result.ts`
- **Fields:**
  - `assignmentId`: ObjectId (References the `Assignment` collection).
  - `questionPaper`: The structured AI output matching the `QuestionPaper` TypeScript interface.
    - `title`: String
    - `subject`: String
    - `duration`: String
    - `totalMarks`: Number
    - `sections`: Array of objects. Each section has a `title`, `instruction`, and an array of `questions`.
      - `Question`: Object containing `text`, `type`, `difficulty` (`easy`, `medium`, `hard`), and `marks`.
  - `createdAt` / `updatedAt`: Timestamps
